## Section 1 — AHIMA Compliance Requirements

### What Makes a Query Compliant

The American Health Information Management Association (AHIMA)
defines standards for compliant physician queries. All queries
generated by our CDI agent must follow these standards.

**A compliant query must be:**
1. **Non-leading** — does not suggest the answer
2. **Based on objective clinical evidence** — cites specific
   lab values, vitals, or findings from the medical record
3. **Open-ended or multiple choice** — gives the physician
   genuine options including "unable to determine" and
   "condition not present"
4. **Clinically relevant** — the question connects to a real
   documentation gap, not fishing for codes
5. **Timely** — asked while the physician can still amend
   documentation (before discharge preferred)

### What Makes a Query Non-Compliant (Prohibited)

**Leading queries (NEVER generate these):**
- "Would you agree the patient has sepsis?"
- "Please document acute kidney injury."
- "The patient's creatinine rise indicates AKI, correct?"
- Any yes/no question that implies the desired answer

**Why compliance matters:** OIG specifically audits CDI programs.
Non-leading queries are a legal requirement, not a best
practice. Leading queries that result in code upgrades can be
viewed as schemes to inflate reimbursement — FCA liability.

## Section 2 — Top 10 CDI Opportunities With Triggers

### 1. Acute Kidney Injury (AKI) — MCC

**Detection trigger:**
- Creatinine rise ≥ 0.3 mg/dL within 48 hours (KDIGO Stage 1)
- OR creatinine ≥ 1.5x baseline within 7 days
- AND note does NOT contain: "acute kidney injury," "AKI,"
  "acute renal failure," "ARF," "acute renal insufficiency"

**Revenue impact:** N17.x is MCC. $3,000-$8,000 per case.

**Query template:**
```
The patient's creatinine rose from [baseline] to [current]
within [timeframe], which meets KDIGO criteria for acute
kidney injury.

Based on your clinical assessment, does the patient have:
□ Acute kidney injury (AKI)
□ Acute on chronic kidney disease
□ Chronic kidney disease progression
□ Creatinine elevation due to other cause: ___________
□ Unable to determine at this time

If AKI is present, please specify the stage and etiology
if known.
```

### 2. Sepsis — MCC

**Detection trigger:**
- ≥ 2 SIRS criteria: temp >38.3°C or <36°C, HR >90,
  RR >20 or PaCO2 <32, WBC >12K or <4K or >10% bands
- AND documented or suspected infection source
- AND note does NOT contain: "sepsis," "septic," "SIRS"

**Revenue impact:** DRG 870 (with MCC) ~$49,690 vs DRG 872
(without MCC) ~$6,931. Difference: ~$42,759.

**Query template:**
```
The patient meets SIRS criteria ([specific criteria met])
and has a documented/suspected [infection source].

Based on your clinical assessment:
□ Sepsis is present
□ Severe sepsis with organ dysfunction (specify: _______)
□ Septic shock
□ SIRS without infection / non-infectious etiology
□ Infection present but does not meet sepsis criteria
□ Unable to determine at this time
```

### 3. Heart Failure Type/Acuity — CC/MCC

**Detection trigger:**
- Note contains "CHF," "heart failure," "HF" without
  type qualifier (systolic/diastolic/HFrEF/HFpEF) or
  acuity qualifier (acute/chronic/acute-on-chronic)
- BNP > 400 pg/mL corroborating

**Revenue impact:** I50.9 (unspecified) = Non-CC. I50.23
(acute on chronic systolic) = MCC. $7,500/case difference.

**Query template:**
```
Heart failure is documented without specification of type
or acuity. BNP is [value].

Please clarify:
Type: □ Systolic (HFrEF)  □ Diastolic (HFpEF)  □ Combined
Acuity: □ Acute  □ Chronic  □ Acute on chronic
□ Heart failure is not present
□ Unable to determine type/acuity at this time
```

### 4. Respiratory Failure — MCC

**Detection trigger:**
- O2 sat < 90% or PaO2 < 60 mmHg
- OR patient on BiPAP, high-flow nasal cannula, or ventilator
- AND note documents "hypoxia" or "respiratory distress"
  but NOT "respiratory failure"

**Revenue impact:** J96.0x is MCC. $4,000-$10,000/case.

### 5. Malnutrition — CC/MCC

**Detection trigger:**
- BMI < 18.5 OR albumin < 3.0 g/dL OR prealbumin < 11 mg/dL
- OR unintentional weight loss > 5% in 30 days
- OR dietitian consult ordered
- AND note does NOT contain "malnutrition"

**Revenue impact:** E43 (severe) = MCC, E44.x (moderate) = CC.
$3,000-$9,000/case. 53% of inpatients malnourished, <5% coded.

### 6. Encephalopathy — MCC

**Detection trigger:**
- Altered mental status documented
- AND metabolic derangement present (hyponatremia, uremia,
  hepatic failure, hypoxia, hyperammonemia)
- AND note says "AMS" or "confusion" but NOT "encephalopathy"

**Revenue impact:** G93.41 = MCC. $4,000-$10,000/case.

### 7. COPD Exacerbation — CC/MCC

**Detection trigger:**
- COPD documented
- AND worsening dyspnea, increased nebulizer/steroid use
- AND note says "COPD" without "exacerbation"

**Revenue impact:** J44.1 = MCC as secondary. $2,000-$5,000.

### 8. Diabetes Complications — CC

**Detection trigger:**
- DM documented without complications
- AND note mentions neuropathy, nephropathy, retinopathy,
  or foot ulcer not linked to diabetes

**Revenue impact:** E11.22 = CC. $1,500-$4,000/case.

### 9. Pressure Ulcer Staging — MCC

**Detection trigger:**
- Wound care notes document stage but physician note says
  only "pressure ulcer" without stage/location

**Revenue impact:** L89.x03 (stage 3) = MCC. $3,000-$8,000.

### 10. Alcohol Withdrawal/Dependence — CC/MCC

**Detection trigger:**
- CIWA protocol ordered or benzodiazepine administered for
  withdrawal management
- AND note documents "alcohol use" or "ETOH" but NOT
  "dependence" or "withdrawal"

**Revenue impact:** F10.231 (withdrawal delirium) = MCC.
$3,000-$7,000/case.

## Section 3 — Query Template Structure

### Standard Structure

```
[HEADER]
Clinical Documentation Query
Date: [date]
Encounter: [encounter_id]
Provider: Dr. [name]

[CLINICAL EVIDENCE — objective data only]
The following objective findings are noted in the medical
record:
- [Lab value 1 with date]
- [Lab value 2 with date]
- [Clinical finding from note with date]

[QUESTION — non-leading, multiple choice]
Based on your clinical assessment, [specific question]:
□ [Option A — the condition is present (with specificity)]
□ [Option B — alternative diagnosis]
□ [Option C — condition not present]
□ [Option D — unable to determine at this time]
□ Other: ___________

[EDUCATIONAL CONTEXT — optional]
Note: [Brief explanation of why this matters for accurate
coding, e.g., "Acute kidney injury documentation enables
accurate severity-of-illness reporting."]

[RESPONSE DEADLINE]
Please respond within 24 hours. If no response is received,
a follow-up query will be sent.
```

### Rules for the Question Section

- Always include "condition not present" as an option
- Always include "unable to determine" as an option
- Always include an "Other" free-text option
- Never phrase as yes/no
- Never use language that suggests the "correct" answer
- Present options in neutral order (not leading with the
  revenue-maximizing answer)

## Section 4 — Workflow Integration

### When CDI Agent Runs

1. **Trigger:** Clinical note signed event (via FHIR
   DocumentReference status change to "current")
2. **Timing:** Concurrent CDI (during admission, before
   discharge). Per ADR-007, proactive CDI is preferred
   over reactive post-discharge review.
3. **Priority:** P0 opportunities (AKI, Sepsis, HF) are
   queried immediately. Lower-priority queries batch daily.

### Query Delivery

1. CDI agent generates query → stored in system database
2. Query displayed in coder review UI (Panel 3 — DRG Impact)
3. Future: direct delivery to physician's EHR inbox via
   FHIR CommunicationRequest or EHR messaging API

### After Physician Responds

- Response documented → coding agent re-analyzes with
  updated documentation
- No response in 24 hours → escalation query sent
- No response in 48 hours → flag for CDI manager review
- Physician disagrees → accept physician's clinical
  judgment; document in audit trail

### Key Principle

The CDI agent identifies documentation gaps. It does NOT
code conditions. It does NOT suggest diagnoses. It asks
the physician to clarify documentation so the coder can
code accurately. The physician's clinical judgment is final.
