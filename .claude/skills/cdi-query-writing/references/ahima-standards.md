# AHIMA Compliance Standards for Physician Queries

## What Makes a Query Compliant

The American Health Information Management Association (AHIMA)
defines standards for compliant physician queries. All queries
generated by our CDI agent must follow these standards.

### A compliant query must be:

1. **Non-leading** — does not suggest the answer
2. **Based on objective clinical evidence** — cites specific
   lab values, vitals, or findings from the medical record
3. **Open-ended or multiple choice** — gives the physician
   genuine options including "unable to determine" and
   "condition not present"
4. **Clinically relevant** — the question connects to a real
   documentation gap, not fishing for codes
5. **Timely** — asked while the physician can still amend
   documentation (before discharge preferred)

### What Makes a Query Non-Compliant (Prohibited)

**Leading queries (NEVER generate these):**
- "Would you agree the patient has sepsis?"
- "Please document acute kidney injury."
- "The patient's creatinine rise indicates AKI, correct?"
- Any yes/no question that implies the desired answer
- Any query that mentions reimbursement or revenue impact
- Any query that tells the physician what to document

**Why compliance matters:** OIG specifically audits CDI programs.
Non-leading queries are a legal requirement, not a best
practice. Leading queries that result in code upgrades can be
viewed as schemes to inflate reimbursement — FCA liability.

### Rules for the Question Section

- Always include "condition not present" as an option
- Always include "unable to determine" as an option
- Always include an "Other" free-text option
- Never phrase as yes/no
- Never use language that suggests the "correct" answer
- Present options in neutral order (not leading with the
  revenue-maximizing answer)
- Never mention revenue, DRG, or reimbursement impact
  in the query text shown to physicians

### Audit Trail

Every query must be logged with:
- encounter_id, cdi_category, target_code, timestamp
- Physician response and response timestamp
- Whether documentation was amended
- Never log clinical content in the audit trail
