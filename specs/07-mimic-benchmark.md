## 1. Purpose

The MIMIC-IV benchmark answers one question:

> When the coding agent analyzes a real hospital discharge summary,
> what percentage of its ICD-10 suggestions match the codes a
> credentialed coder actually assigned?

This measures **precision** (acceptance rate) — the fraction of
AI suggestions that are correct per ground truth. It is the metric
CFOs and HIM Directors will ask about in procurement reviews.

Target (Phase 1 exit criteria, DESIGN-000 §4):
- **Acceptance rate (precision) ≥ 0.70** (70%)
- **Recall rate ≥ 0.55** (55% of gold-standard codes found)

## 2. Data Sources

### 2.1 MIMIC-IV

MIMIC-IV (Medical Information Mart for Intensive Care, version 2.2)
is a large, freely available de-identified EHR dataset from Beth
Israel Deaconess Medical Center (2008–2019).

Access: PhysioNet (requires credentialed researcher registration)
URL: https://physionet.org/content/mimiciv/2.2/

**Required tables:**

| File | Module | Contents |
|------|--------|----------|
| `hosp/diagnoses_icd.csv` | MIMIC-IV hosp | ICD codes per admission |
| `note/discharge.csv` | MIMIC-IV-Note | Discharge summary text |

**Key fields:**

`diagnoses_icd.csv`:
- `hadm_id` — hospital admission ID (join key)
- `subject_id` — patient ID (not used in benchmark)
- `seq_num` — code sequence (1 = principal diagnosis)
- `icd_code` — raw ICD code (no dots)
- `icd_version` — 9 or 10 (we filter to 10 only)

`discharge.csv`:
- `hadm_id` — join key
- `note_type` — "Discharge summary" (filter to this type)
- `text` — full discharge summary text

### 2.2 ICD-10 Scope

Only admissions coded with `icd_version=10` are included.
These are post-2015 admissions (after ICD-10 transition on
October 1, 2015).

Expected sample pool: ~20,000–30,000 admissions in MIMIC-IV v2.2.

## 3. Data Structures

```python
class MimicAdmission(BaseModel):
    hadm_id: str
    discharge_note: str          # full discharge summary text
    gold_codes: list[str]        # all ICD-10 codes, formatted with dots
    principal_dx: str            # seq_num=1 code (with dots)
    total_code_count: int        # len(gold_codes)

class BenchmarkResult(BaseModel):
    hadm_id: str
    suggested_codes: list[str]   # from coding agent
    gold_codes: list[str]        # from MIMIC ground truth
    exact_matches: list[str]     # suggested ∩ gold (exact string)
    parent_matches: list[str]    # 3-char prefix matches (lenient)
    precision: float             # exact: len(exact_matches)/len(suggested)
    recall: float                # exact: len(exact_matches)/len(gold)
    is_degraded: bool            # agent returned DegradedResult

class BenchmarkReport(BaseModel):
    run_date: str                # ISO format
    sample_size: int             # admissions analyzed
    valid_runs: int              # non-degraded results
    mean_precision: float        # primary metric (acceptance rate)
    mean_recall: float
    mean_f1: float
    meets_target: bool           # mean_precision >= 0.70
    degraded_count: int          # agent failures
    results: list[BenchmarkResult]
```

## 4. Matching Algorithm

### 4.1 Code Normalization

MIMIC stores codes without dots (`I509` not `I50.9`).
The coding agent returns codes with dots.

Normalize all codes before comparison:
```python
def normalize_code(raw: str) -> str:
    """Remove dots, uppercase. 'I50.9' → 'I509'."""
    return raw.replace(".", "").upper().strip()
```

### 4.2 Exact Match

```python
def exact_match(suggested: list[str], gold: list[str]) -> list[str]:
    norm_gold = {normalize_code(c) for c in gold}
    return [c for c in suggested if normalize_code(c) in norm_gold]
```

### 4.3 Parent Match (Lenient)

Matches on the first 3 characters (ICD-10 category level).
Used for secondary reporting only — not for the acceptance rate target.

```python
def parent_match(suggested: list[str], gold: list[str]) -> list[str]:
    gold_prefixes = {normalize_code(c)[:3] for c in gold}
    return [c for c in suggested if normalize_code(c)[:3] in gold_prefixes]
```

### 4.4 Precision and Recall

```python
def precision(suggested: list[str], gold: list[str]) -> float:
    """Fraction of suggestions that match gold (acceptance rate)."""
    if not suggested:
        return 0.0
    return len(exact_match(suggested, gold)) / len(suggested)

def recall(suggested: list[str], gold: list[str]) -> float:
    """Fraction of gold codes found in suggestions."""
    if not gold:
        return 0.0
    return len(exact_match(suggested, gold)) / len(gold)
```

## 5. Sample Size and LLM Cost

### 5.1 Pilot Run (default)

| Parameter | Value |
|-----------|-------|
| Sample size | 100 admissions |
| LLM tokens/note | ~3,000 input + ~500 output |
| Cost (Sonnet 4.6) | ~$0.015/note × 100 = ~$1.50 |
| Runtime | ~45–90 minutes (rate limit: 60 req/min) |

### 5.2 Full Benchmark

| Parameter | Value |
|-----------|-------|
| Sample size | 1,000 admissions |
| Cost estimate | ~$15–25 |
| Runtime | ~8–16 hours |
| Statistical significance | p < 0.001 for ±2% CI |

### 5.3 Rate Limiting

MIMIC-IV discharge notes can be long (2,000–10,000 tokens).
The benchmark runner sleeps 1 second between LLM calls to stay
within Anthropic API rate limits. Configurable via `BENCHMARK_DELAY_S`
environment variable.

### 6.1 Tests That Always Run (no MIMIC data required)

| Test | Target |
|------|--------|
| `test_acceptance_rate_exceeds_70_percent` | mean_precision ≥ 0.70 |
| `test_recall_rate_exceeds_55_percent` | mean_recall ≥ 0.55 |
| `test_principal_dx_found_in_majority` | principal dx in suggestions ≥ 80% |
| `test_no_phi_in_benchmark_report` | PHI fields absent from report JSON |

Separate from pytest — invoked directly for full benchmarks:
```bash
uv run python -m src.benchmarks.mimic_benchmark --sample 100
```

Writes results to `data/mimic/benchmark_results/` as JSON.

## 7. Compliance Notes

### 7.1 PHI Handling

MIMIC-IV is de-identified under HIPAA Safe Harbor. However:
- `discharge_note` text is NOT logged (clinical content)
- `hadm_id` and `subject_id` are identifiers, not PHI per Safe Harbor,
  but are excluded from log messages to maintain hygiene
- `benchmark_results/` is gitignored — never committed
- `discharge.csv` and `diagnoses_icd.csv` are gitignored

### 7.2 Ground Truth Limitations

MIMIC-IV codes reflect actual coder assignments which may include:
- Coding errors (human coders ~85–95% consistent per literature)
- ICD-9 to ICD-10 conversion artifacts
- Institution-specific coding conventions

A precision of 70% against MIMIC ground truth does not mean
30% of suggestions are wrong — some may be improvements over
the original human coding.
